# Evaluation of LLMs and Olive on DSBench

This repository evaluates Large Language Models on the [DSBench (Data Analysis)](https://github.com/LiqiangJing/DSBench/tree/main/data_analysis) dataset. It systematically tests LLM capabilities in data science code generation by generating responses to complex data science questions and evaluating them using LLM-based judge.

We have used a subset of **DSBench (Data Analysis)** for this experiment, comprising 22 challenges (236 questions).


## Quick Start

### 1. Install Conda environment and Dependencies
```bash
conda create -n dsbench python=3.10
pip install -r requirements.txt
```
### 2. Add your API keys to .env file
```bash
OPENAI_API_KEY=<your_openai_api_key_here>
OPENROUTER_API_KEY=<your_openrouter_api_key_here>
USE_OPENROUTER=true
```
### 3. Run Jupyter Notebooks
Set paths for dataset properly and run respective LLM notebook

### 4. Run Evaluation Script
```bash
cd data_analysis
python compute_answer.py
python show_result.py
```


## How to evaluate GPT on data analysis tasks
1. Ensure the correct dataset path, which is present in `data_analysis`.
3. Set your keys/env variables against respective notebooks.
4. Run the notebook to evaluate the performance. The results will be saved into the path `./save_process/{model}/`
5. Show the evaluation results:

   ```
   python compute_answer.py
   ```
   Determine whether the answers generated by the model are correct and save the result into a JSON file `results.json`.
   
   ```
   python show_result.py
   ```
   Show the performance of the model based on the `results.json` and files generated from the evaluation process.

## Features

- **Multi-Provider LLM Support**: OpenAI, Anthropic Claude, Google Vertex AI, OpenRouter
- **LLM-as-Judge Evaluation**: Uses LLM to score and evaluate responses


## Project Structure

```
dsbc_evaluation/
├── data_subset_olive/         # Dataset Dir (Subset of DSBench Data_Analysis)
├── save-process/              # Evaluation module
├── compute_answer.py          # Python script for computing evaluation
├── eval_Claude-Opus-4.6.ipynb      # Claude Opus 4.5 Notebook
├── eval_GLM-5.ipynb                # GLM 5 Notebook
├── eval_GPT-5.2.ipynb              # GPT 5.2 Notebook
└── show_result.py             # Python script for showing results
├── save-process/              # Results directory
│   ├── <model_name>/          # Model Name based sub-directory
    .
    │
└── requirements.txt           # Python dependencies

```





## Output Files

- **Processed Data**: `temp/processed_data/{model_name}_{uuid}.csv`
- **Evaluated Data**: `temp/evaluated_data/evaluated_{model1}_{model2}_{uuid}.csv`
- **Results**: `temp/results/evaluation_{filename}.json`
